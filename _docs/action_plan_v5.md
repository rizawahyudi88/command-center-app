**Versi Arsitektur:** 5.0 (Zero-Cost Serverless)
**Technical Architect:** Riza Fazrul Wahyudi, S.Kom

| Fase / Durasi | Detail Aktivitas (Action) | Output (Hasil Teknis) | Outcome (Dampak Bisnis) | Indikator Terukur (KPI) |
| :--- | :--- | :--- | :--- | :--- |
| **Sprint 1: Fondasi Infrastruktur & Autentikasi**<br>*(Minggu 1)* | 1. Setup repositori Next.js dengan arsitektur React Server Components (RSC).<br>2. Integrasi Tailwind CSS dengan tema *Matte Obsidian* & *Swiss-style*.<br>3. Konfigurasi proyek Supabase, inisialisasi tabel SQL (DDL), dan pengaturan *Role-Based Access Control* (RBAC). | Lingkungan *development* terkonfigurasi. Sistem autentikasi berjenjang (Admin, Pimpinan, Staf) berfungsi di antarmuka awal. | Keamanan data dasar terjamin dan landasan sistem berbasis *zero-cost* terbentuk, memungkinkan pendelegasian akses yang terkontrol. | 100% rute halaman sensitif terlindungi secara otomatis oleh sesi Supabase Auth. |
| **Sprint 2: Core Document Engine & File Storage**<br>*(Minggu 2)* | 1. Konfigurasi kredensial *Service Account* Google Cloud untuk integrasi Drive API.<br>2. Pembuatan API Route `/api/generate` di Next.js yang menjalankan mesin `docxtemplater`.<br>3. Eksekusi pengujian injeksi variabel JSON ke file *template* `.docx` kosong. | API aktif mengonversi *payload* menjadi file Microsoft Word baru dan menyimpannya langsung ke direktori spesifik di Google Drive. | Membuktikan efektivitas arsitektur dalam mencetak naskah dinas tanpa merusak margin atau stempel instansi sedikit pun. | *Formatting drift* (pergeseran tata letak) 0%, kecepatan waktu respons API di bawah 5 detik. |
| **Sprint 3: Modul Command Center & Integrasi AI**<br>*(Minggu 3)* | 1. Pembuatan *dashboard* matriks tugas dan *form* pendelegasian (Administrasi & Operasional).<br>2. Pembuatan form pelaporan lapangan (unggah foto & input poin giat).<br>3. Integrasi *prompt* sistem ke Gemini API untuk merangkai Laporan Hasil Kegiatan (LHK). | Antarmuka interaktif yang menampilkan penugasan berstatus aktif. Asisten AI berhasil mengonversi teks acak menjadi paragraf naskah resmi. | Mengakselerasi penyusunan laporan operasional lapangan secara drastis; staf tidak perlu mengetik dari nol. | Gemini API mengembalikan narasi sesuai EYD dalam waktu di bawah 4 detik per permintaan. |
| **Sprint 4: Workflow Pengawasan & UI Polish**<br>*(Minggu 4)* | 1. Implementasi *State Machine* (`DRAFTING` ➔ `FINALIZED`) dan pencatatan tabel `audit_logs`.<br>2. Mengamankan pratinjau PDF di UI melalui fitur *Signed URL* Google Drive.<br>3. Pembuatan antarmuka panel samping untuk kolom catatan revisi pimpinan. | Alur *approval* berjalan berjalan sempurna. Pimpinan dapat menolak/menyetujui naskah, dan sistem otomatis membuat dokumen versi terbaru (V2). | Transparansi pengawasan berjenjang terjamin, riwayat dokumen tetap utuh, dan file tidak dapat dibocorkan melalui tautan publik. | 100% jejak revisi tercatat di *database*, URL PDF pratinjau mati secara otomatis dalam 15 menit. |
| **Sprint 5: UAT, Optimasi, & Deployment**<br>*(Minggu 5)* | 1. Pelaksanaan *User Acceptance Testing* (UAT) dengan menyimulasikan skenario penugasan lengkap.<br>2. Perbaikan *bug* antarmuka dan optimasi *database query*.<br>3. Pemindahan konfigurasi ke *production environment* dan rilis operasional di Vercel. | Aplikasi v1.0 siap diakses (*Live*) dengan standar performa dan keamanan tinggi. Dokumentasi teknis rampung. | Sistem otomatisasi siap digunakan untuk operasional harian institusi secara penuh tanpa adanya tagihan *hosting* server. | Terselesaikannya seluruh tiket uji coba (0 *critical bugs*) dan aplikasi lulus standar akses institusi. |
